专注软件开发需求 - 项目总览

本文档用于从“整体链路”的角度解释项目结构、页面/状态/接口如何串起来，以及各功能的实现入口在哪里。
入口目录：src/（uni-app + Vue3）与 server/（Node/Express）。
建议使用 VSCode 打开本 HTML 并点击文件链接快速跳转。

目录

整体架构 页面与路由 登录 专注计时 职级与晋升 待办 + 计时 好友与通知 自习室 白噪音 网络请求封装

整体架构

前端
uni-app + Vue3 页面在 src/pages/；Pinia 状态在 src/stores/；请求封装在 src/utils/request.ts
后端
Express 服务在 server/index.js；可接 Supabase（环境变量配置后启用）；静态托管目录 server/public/（用于音频/职级头像）
数据库
Supabase：users / weekly_stats / focus_sessions / friends / friend_invites 等（schema 参考 src/app/database/schema.sql）

登录（微信一键登录链路）

核心流程：小程序 wx.login 获取 code → 后端 /api/wechat/login 换 openid → 绑定/创建用户 → 返回 token。

关键文件

  • src/pages/auth/index.vue
  • src/api/user.ts
  • src/utils/request.ts
  • server/index.js
sequenceDiagram participant FE as 小程序 participant API as server/index.js participant WX as 微信API participant DB as Supabase FE->>WX: wx.login() WX-->>FE: code FE->>API: POST /api/wechat/login {code} API->>WX: jscode2session(code) WX-->>API: openid, session_key API->>DB: upsert users(wechat_openid) API-->>FE: {token, user} FE->>FE: Storage 写入 token/user

专注计时（倒计时/正计时 + 严格/温和规则）

计时状态统一由 timerStore 管理。首页选择时长/事项/模式后启动计时并跳转到专注页；待办计时会启动正计时模式。

关键文件

  • src/stores/timer.ts
  • src/pages/home/index.vue
  • src/pages/timer/index.vue
sequenceDiagram participant Home as 首页 participant Store as timerStore participant Timer as 专注页 participant API as /api/focus/end Home->>Store: startFocus(duration, category, mode) Home->>Timer: navigateTo /pages/timer Timer->>Store: tick() 每秒 Timer->>Store: pause/resume/stop Timer->>API: syncFocusEnd(duration_minutes, points, rank_after)

严格/温和退出规则通过专注页的 onHide/onUnload/onBackPress 触发 timerStore.registerQuit() 实现，严格模式退出会清零积分并结束本次。

职级与晋升（名称/阈值/头像）

职级配置统一在 RANK_CONFIG；晋升弹窗由专注页展示，展示旧职级→新职级的头像、名称、积分增量。

关键文件

  • src/types/index.ts
  • src/pages/timer/index.vue

职级头像建议放到后端静态目录：server/public/ranks/*.png，通过 https://tomatoclock.onrender.com/static/ranks/xxx.png 访问，避免小程序包体积膨胀。

待办清单 + 任务计时

待办数据在 todoStore（本地持久化），每个待办可直接进入专注页并开始正计时，结束后把本次耗时回写到待办中。

关键文件

  • src/stores/todo.ts
  • src/components/TodoChecklist.vue
  • src/pages/home/index.vue
  • src/pages/timer/index.vue
sequenceDiagram participant UI as TodoChecklist participant Todo as todoStore participant Timer as timerStore participant Page as 专注页 UI->>Todo: startTodoFocus(todoId) UI->>Timer: startCountup(category, mode) UI->>Page: navigateTo /pages/timer Page->>Timer: tick() Page->>Todo: finishActiveFocus(elapsedSeconds)

好友系统 + 订阅消息通知

好友系统采用“邀请码→发起申请→对方同意→双方成为好友”。发送订阅消息由后端触发（需要用户在小程序端授权订阅）。

关键文件

  • src/pages/friends/index.vue
  • src/api/friends.ts
  • server/index.js
sequenceDiagram participant A as 申请人 participant B as 被申请人 participant API as 后端 A->>API: POST /api/friends/invite {invite_code} API-->>B: 订阅消息：好友申请 B->>API: POST /api/friends/invites/:id/accept API-->>A: 订阅消息：申请通过

“提醒通知”开关通过前端控制是否触发 requestSubscribeMessage，实现用户可控的订阅授权行为。

自习室（创建/加入/心跳/离开）

自习室为内存房间（Map）实现：join 创建/加入、ping 更新心跳并返回成员列表、leave 离开并在空房间时销毁。

关键文件

  • src/pages/study-room/index.vue
  • server/index.js

白噪音（静态托管 + 小程序播放）

音频资源建议放在后端静态目录：server/public/audio/*.mp3，通过 /static/audio/*.mp3 访问。小程序端播放采用 downloadFile → tempFilePath（更稳定）并带直连兜底。

关键文件

  • server/index.js（静态托管 /static）
  • src/pages/timer/index.vue（initAudio）

如出现“下载失败”，通常需要在微信公众平台配置 downloadFile 合法域名（tomatoclock.onrender.com）。

网络请求封装（baseURL/重试/错误提示）

request.ts 统一注入 token、拼接 baseURL、处理 401 跳转登录、Render 冷启动超时重试、以及 DNS 解析失败提示跳转“开发配置页”。

  • src/utils/request.ts
  • src/pages/dev-api/index.vue
